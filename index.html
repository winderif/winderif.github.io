First android studio & gradle

當初接觸 android 時就是在 eclipse 上開發，習慣之後還真的覺得很好用，直到最近發現 android 也有自己的一套開發工具－ Android Studio。於是就想來試試看，因為各大討論區都覺得早晚都要熟悉 AS，畢竟已經成為官方指定的開發工具了，所以遲早還是要把 eclipse 專案轉移到 AS 上開發，所以就趁早來熟悉吧！

開始 Android Studio

官方網站上就可以下載 AS 的安裝檔，不過安裝 AS 之前要先裝好 JDK，如果沒裝的話也沒關係，AS 安裝過程也會引導你裝上 JDK。網路上已經有很多教學就不多說了。

從 Eclipse 匯出專案到 Android Studio 上

原本在 eclipse 上開發的專案 code，現在也可以輕鬆的轉移到 AS。

  • 在 eclipse 上，選擇 "File" -> "Export"
  • 點選 "Android" -> "Generate gradle build files"
  • 再選擇要匯出的專案，會產生 ".Gradle" 檔案
  • 在 Android Studio 上，第一個畫面選擇 "Import Non-Android Studio Project" -> "Import project from external model" -> "Gradle"
  • 接著，選擇剛才匯出的 gradle project

後記心得

Gradle 加入失敗

轉移到 AS 遇到第一個最大問題就是－Gradle 是什麼？還真的是讓我有點頭痛。Gradle 可以幫你自動的 import 第三方 library 到專案，很像以前在 eclipse 上加入 external JAR。不熟悉 gradle 真的讓我搞很久，尤其是 gradle sync 之後出現錯誤訊息，完全不知道發生什麼事，不過幸好有 google 大神，嘗試開了新的專案測試 gradle，成功後再回到專案弄一弄就沒問題了。

設定github

版本控制當然還是必要呀，在 AS 上也有完整的 VCS 支援，熟悉之後用起來大同小異，反正知道 git 怎麼用到哪都可以。

AS 的優點

  • 比起 eclipse，他不需要每次打開都載入所有的專案 code 佔用電腦資源，每個 project 開一個視窗，project 下還可以有多個 module。
  • 搜尋功能我想是 AS 最強大的地方，在 eclipse 上也有類似的功能，但有限定字串的順序，AS 則是只要包含字串的所有資訊都會列出來給你，真的是用過之後回不去了。

First Markdown!!

最近開始有再用 github 寫點玩具，就想說不知道有什麼方法可以在 github 上寫 blog，google一堆資料，發現很多都是用 Octopress ，不過需要先安裝 ruby 在 windows 上感覺有點麻煩，後來發現 Hexo 也是 blog framework ，而且不是用 ruby 是用 node.js，就趕緊嘗試看看。

開始Hexo

  • 安裝 Node.js

  • 安裝 Hexo

  • 建立 blog 資料夾

  • 進到資料夾然後打指令

  • 產生 blog

  • 讓 blog 可以在 localhost 預覽

  • 上傳 blog 到 Github 上 (詳細就不說了，直接參考置底連結)

    遇到困難

    出現 events.js:85 的錯誤

    解決方法：直接在git shell上執行hexo d

    git branch master 和 gh-pages 的差別

    一開始 deploy 到 gh-pages 想說 repo 上怎麼沒有反應，連結也一直404，後來才發現原來使用 branch gh-pages 的意義不太一樣。簡單來說：

  • master: 當 deploy 到 master 分支，你就可以直接在 yourname.github.io 看到你的網頁
  • gh-pages: 假設你的 repo 名稱叫做 project，你想要在這個 project 建立一個網頁，那你就必須 deploy 到 gh-pages 分支，然後你就可以在 yourname.github.io/project 看到你的網頁
