First Markdown!!

最近開始有再用 github 寫點玩具，就想說不知道有什麼方法可以在 github 上寫 blog，google一堆資料，發現很多都是用 Octopress ，不過需要先安裝 ruby 在 windows 上感覺有點麻煩，後來發現 Hexo 也是 blog framework ，而且不是用 ruby 是用 node.js，就趕緊嘗試看看。

開始Hexo

  • 安裝 Node.js
  • 安裝 Hexo
    npm install hexo -g
  • 建立 blog 資料夾
  • 進到資料夾然後打指令
    hexo init
    npm install
  • 產生 blog
    hexo g
  • 讓 blog 可以在 localhost 預覽
    hexo s
  • 上傳 blog 到 Github 上 (詳細就不說了，直接參考置底連結)

    遇到困難

    出現 events.js:85 的錯誤

    解決方法：直接在git shell上執行hexo d

    git branch master 和 gh-pages 的差別

    一開始 deploy 到 gh-pages 想說 repo 上怎麼沒有反應，連結也一直404，後來才發現原來使用 branch gh-pages 的意義不太一樣。簡單來說：
  • master: 當 deploy 到 master 分支，你就可以直接在 yourname.github.io 看到你的網頁
  • gh-pages: 假設你的 repo 名稱叫做 project，你想要在這個 project 建立一個網頁，那你就必須 deploy 到 gh-pages 分支，然後你就可以在 yourname.github.io/project 看到你的網頁
